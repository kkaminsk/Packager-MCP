# WinRAR 7.13.0 - Intune Detection Rules

## Overview
This document provides detection rules for WinRAR 7.13.0 deployment via Microsoft Intune. Two detection methods are provided: file-based and registry-based.

---

## Option 1: File Version Detection (Recommended)

Detects WinRAR by checking the version of `WinRAR.exe`.

```json
{
  "@odata.type": "#microsoft.graph.win32LobAppFileSystemDetection",
  "path": "C:\\Program Files\\WinRAR",
  "fileOrFolderName": "WinRAR.exe",
  "check32BitOn64System": false,
  "detectionType": "version",
  "operator": "greaterThanOrEqual",
  "detectionValue": "7.13.0.0"
}
```

### Intune Portal Configuration

|---------|-------|

| **Path** | `C:\Program Files\WinRAR` |
| **File or folder** | `WinRAR.exe` |
| **Detection method** | File version |
| **Operator** | Greater than or equal to |
| **Value** | `7.13.0.0` |
| **Associated with a 32-bit app on 64-bit clients** | No |

### PowerShell Detection Script
```powershell
# File version detection script
# Exit 0 if detected, Exit 1 if not detected
$FilePath = 'C:\Program Files\WinRAR\WinRAR.exe'
$RequiredVersion = '7.13.0.0'

if (Test-Path -Path $FilePath) {
    $FileVersion = (Get-Item $FilePath).VersionInfo.FileVersion
    if ($FileVersion) {
        # Clean version string (remove extra info after space)
        $FileVersion = ($FileVersion -split ' ')[0]
        if ([version]$FileVersion -ge [version]'7.13.0.0') {
            Write-Host "Detected: Version $FileVersion meets requirement"
            exit 0
        }
        Write-Host "Not detected: Version $FileVersion does not meet requirement $RequiredVersion"
        exit 1
    }
    Write-Host "Not detected: Could not read version info from $FilePath"
    exit 1
}

Write-Host "Not detected: File not found at $FilePath"
exit 1
```

## Option 2: Registry Detection

Detects WinRAR by checking the DisplayVersion in the uninstall registry key.

### Intune Graph API JSON
```json
{
  "@odata.type": "#microsoft.graph.win32LobAppRegistryDetection",
  "keyPath": "HKEY_LOCAL_MACHINE\\SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Uninstall\\WinRAR archiver",
  "valueName": "DisplayVersion",
  "check32BitOn64System": false,
  "detectionType": "version",
  "operator": "greaterThanOrEqual",
  "detectionValue": "7.13.0"
}
```

### Intune Portal Configuration
| Setting | Value |
|---------|-------|
| **Rule type** | Registry |
| **Key path** | `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\WinRAR archiver` |
| **Value name** | `DisplayVersion` |
| **Detection method** | Version comparison |
| **Operator** | Greater than or equal to |
| **Value** | `7.13.0` |
| **Associated with a 32-bit app on 64-bit clients** | No |

### PowerShell Detection Script
```powershell
# Registry version detection script
# Exit 0 if detected, Exit 1 if not detected
$RegPath = 'HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\WinRAR archiver'
$ValueName = 'DisplayVersion'
$RequiredVersion = '7.13.0'

try {
    $Value = (Get-ItemProperty -Path $RegPath -Name $ValueName -ErrorAction Stop).$ValueName
    if ($Value) {
        if ([version]$Value -ge [version]'7.13.0') {
            Write-Host "Detected: Version $Value meets requirement"
            exit 0
        }
        Write-Host "Not detected: Version $Value does not meet requirement $RequiredVersion"
        exit 1
    }
} catch {
    # Value doesn't exist or can't be read
}

Write-Host "Not detected: Registry value not found"
exit 1
```

## Recommendations

1. **File detection** is recommended as the primary method - it directly verifies the executable exists and meets the version requirement.
2. **Registry detection** is a good fallback and is commonly used for applications that create uninstall registry keys.
3. For production deployments, test detection rules in a lab environment first.
4. Consider using the PowerShell script method for more complex detection scenarios.
